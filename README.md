Este projeto consiste no desenvolvimento e implementação completa de um **SGBD Relacional** para um supermercado. O sistema integra operações críticas (Vendas, Estoque, Compras e RH), substituindo decisões intuitivas por uma gestão baseada em dados estruturados.

## 📌 Destaques Técnicos e Entregáveis

- **Arquitetura SQL (DDL/DML):** Tradução de modelos lógicos para uma estrutura física robusta, com foco em normalização e integridade referencial para eliminação de redundâncias[cite: 1, 2].
- **Business Intelligence com Views:** Consultas complexas que geram relatórios estratégicos de margem de lucro, desempenho de fornecedores e produtividade[cite: 1, 2].
- **Data Analytics com Python:** Integração do banco de dados com bibliotecas como **Pandas, Matplotlib e Seaborn** para criação de dashboards interativos e visualizações de gestão[cite: 1, 2].
- **Sistema de Alerta Antecipado:** Lógica de monitoramento de estoque crítico para prevenção de rupturas e otimização do capital de giro[cite: 1, 2].

## 📊 Análises de Desempenho 360º

O sistema foi projetado para oferecer uma visão completa do negócio, incluindo:
* **Sazonalidade e Lucratividade:** Identificação de produtos com maior giro para renegociação com parceiros comerciais[cite: 1, 2].
* **Gestão de Equipe:** Métricas de produtividade integradas ao banco de dados[cite: 1, 2].
* **Margem de Lucro:** Views específicas para análise detalhada por categoria de produto[cite: 1, 2].

## 🛠️ Stack Tecnológica
- **Banco de Dados:** SQL 
- **Linguagem de Análise:** Python
- **Bibliotecas:** Pandas, Matplotlib, Seaborn
- **Ferramentas:** Geração de relatórios estratégicos e Dashboards

## 📈 Conclusão
O projeto demonstrou a viabilidade técnica de unir arquitetura de dados com análise visual, garantindo o suporte necessário para o crescimento sustentável de um negócio de varejo.
